# Lanterne — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Chrome extension that replaces the new tab page with a warm, customizable personal command center featuring a clock/weather dashboard, daily ritual tracker, and page/clip scrapbook.

**Architecture:** Chrome Manifest V3 extension using `chrome_url_overrides` for new tab. Pure vanilla HTML/CSS/JS — no framework, keeping the bundle tiny and fast. All data persisted in `chrome.storage.local` for offline-first behavior. Optional weather API calls via `fetch`. Modular JS files per feature (clock, rituals, scrapbook, settings).

**Tech Stack:** Chrome Extension (Manifest V3), vanilla HTML/CSS/JS, chrome.storage.local, Open-Meteo API (free, no key), Lucide icons (SVG, MIT license), Google Fonts (Inter + optional serif), Unsplash/Pexels for backgrounds (license-free).

Uses browser geolocation API to get coordinates, then fetches from Open-Meteo (free, no API key):

```
https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current_weather=true
```

Shows temperature and weather icon (using Lucide SVG icons mapped to WMO weather codes). Caches result for 30 minutes in storage to avoid excessive calls.

```js
export async function initWeather(container) {
  const cached = await getCachedWeather();
  if (cached) { render(container, cached); return; }
  // ... geolocation + fetch + cache + render
}
```
